import org.roosti.*;

"roosti.org" is a domain name for package names of open-sourced Java libraries.

When open-sourcing a new small Java library as a person or a small team, naming its Java package is a problem. Internet domain names are used for Java packages.

  • Everyone does not have their own domain name. Even if they have, maintaining domain names may not be the best solution as their lifetimes depend.
  • Domain names of source code hosting services (such as github.com and sf.net) are not always the best solution also due to their lifetimes.
  • Purchasing a new domain name for each library is not very reasonable.

"roosti.org" intends to be a co-managed hosting of a Java package namespace in the future. It is yet owned/managed by a single person for the time being, though.

Q & A

What is "roosti.org"?

  • "roosti.org" is hosting a Java package namespace to reserve.

What is NOT "roosti.org"?

  • "roosti.org" is NOT hosting source code.
  • "roosti.org" is NOT hosting binary distribution.
  • "roosti.org" is NOT hosting web pages.

Is "roosti.org" to be maintained farther into the future?

Not guaranteed unfortunately. The owner may retire, may be bored, or may give up. Assume "roosti.org" is still just a personal domain name which may be accepting requests from others. Co-owners may help!

Even if the owner gives up, will try calling for successors if possible.

Can I join this project as a co-owner?

Co-owners are not publicly called for at the moment. File a GitHub Issue if you are interested, and you are sure the owner knows you.

Can I register a package name for a person or an organization, not for a project?

It is not "prohibited", but the owner does not welcome such registrations. "roosti.org" is not for coffee roasters, but for roastees.

What does "roosti" mean?

It comes from "roost" + "roasty" with a tone of a suffix "-ee".

How is "roosti" pronounced?

Like "roost-ee".
